    Лабораторная работа 2. Веб-сервис «Список отелей»

    Цель работы

    Реализовать веб-сервис для управления отелями со следующим функционалом:

    • Список отелей с учетом названия, владельца, адреса, описания, типов номеров, стоимости, вместимости и удобств
    • Регистрация новых пользователей
    • Просмотр и резервирование номеров, редактирование и удаление своих резервирований
    • Отзывы к номерам (период проживания, текст, рейтинг 1–10, информация о комментаторе)
    • Администратор может заселять и выселять пользователей через Django-admin
    • Таблица постояльцев отеля за последний месяц в клиентской части

    Выполненные задачи

    1. Структура проекта

    Проект построен на Django 5.0 и включает:

    • django_project_shapovalov — конфигурация Django-проекта
    • hotels_app — основное приложение для работы с отелями

    2. Модели данных

    Реализованы модели в hotels_app/models.py:

    Модель Описание
    Hotel Отель: название (name), владелец (owner — FK на User), адрес (address), описание (description)
    RoomType Тип номера: название и описание
    Room Номер: связь с отелем и типом, номер комнаты (number), цена (price), вместимость (capacity), удобства (amenities)
    Reservation Бронирование: пользователь, комната, даты (date_from, date_to), статус (reserved, checked_in, checked_out, cancelled)
    Review Отзыв: связь с резервированием и комнатой, пользователь, рейтинг (1–10), текст, период проживания (period_from, period_to)

    Все перечисленные требования по данным учтены: название отеля, владелец, адрес, описание, типы номеров, стоимость, вместимость, удобства.

    3. Регистрация пользователей

    • Реализован view register с использованием UserCreationForm
    • Страница регистрации: /register/
    • После успешной регистрации пользователь автоматически авторизуется и перенаправляется на список отелей
    • Используется стандартная аутентификация Django (django.contrib.auth)

    4. Просмотр и резервирование номеров

    • Список отелей (/) — карточки отелей с названием, адресом, описанием
    • Детальная страница отеля (/hotel/<id>/) — информация об отеле и список номеров
    • Список номеров (/rooms/) — RoomListView с фильтрацией по названию отеля и типу номера, пагинация (6 номеров на страницу)
    • Детальная страница номера (/room/<id>/) — тип, цена, вместимость, кнопка «Забронировать»
    • Резервирование (/reserve/) — форма с выбором комнаты и дат, проверка пересечения дат перед созданием брони

    5. Редактирование и удаление резервирований

    • Мои брони (/my_reservations/) — таблица резервирований пользователя
    • Редактирование (/reservation/<id>/edit/) — изменение комнаты и дат с проверкой пересечений
    • Отмена (/reservation/<id>/cancel/) — подтверждение и удаление резервирования

    Редактировать и отменять можно только свои резервирования.

    6. Отзывы к номерам

    • Сохраняются: период проживания (period_from, period_to), текст (text), рейтинг 1–10, связь с пользователем и комнатой
    • Добавление отзыва после выселения (/reservation/<id>/review/) — только для завершенных бронирований и только один отзыв на бронирование
    • Альтернативный способ: /room/<id>/review/ — добавить отзыв к комнате
    • На странице номера выводится список отзывов с автором, рейтингом и текстом
    • Форма ReviewForm с валидацией рейтинга (min=1, max=10)

    7. Заселение и выселение в Django-admin

    • Модели Hotel, RoomType, Room, Reservation, Review зарегистрированы в admin.py
    • В Django-admin можно изменять статус резервирования: checked_in (заселение) и checked_out (выселение)
    • Для администраторов также доступны кнопки «Заселить» и «Выселить» на странице /my_reservations/ (при is_staff)

    8. Таблица постояльцев за последний месяц

    • Страница /guests_last_month/ — таблица с полями:
    • Пользователь
    • Отель
    • Номер
    • Период (даты проживания)
    • Выводятся резервирования, пересекающиеся с последними 30 днями (date_to >= today - 30 и date_from <= today)

    9. Клиентская часть

    • Используется Bootstrap 5.3 для стилей
    • Базовый шаблон templates/base.html с навигацией: Отели, Номера, Постояльцы
    • Отдельные CSS для гостей, комнат и регистрации
    • Шаблоны: register.html, hotel_list.html, hotel_detail.html, room_list.html, room_detail.html, reservation_form.html, my_reservations.html, reservation_cancel.html, review_form.html, guests_last_month.html, reservation_error.html
